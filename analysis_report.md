# Анализ проекта Solar Observations

## Обзор архитектуры

Проект представляет собой Phoenix-приложение для автоматического слежения за Солнцем с использованием экваториальной монтировки SkyWatcher. Основные компоненты:

1. **MountWorker** (`lib/solar_observations/workers/mount_worker.ex`) - GenServer для управления монтировкой через последовательный порт
2. **SolarCalculator** (`lib/solar_observations/astronomy/solar_calculator.ex`) - расчет положения Солнца
3. **SolarTracker** (`lib/solar_observations/astronomy/solar_tracker.ex`) - основной модуль для слежения
4. **MountCoordinates** (`lib/solar_observations/astronomy/mount_coordinates.ex`) - преобразование координат

## Выявленные проблемы

### 1. Проблемы с стартовой позицией воркера

**Проблема**: В `application.ex` воркер запускается автоматически при старте приложения, но:

- Нет проверки успешности подключения к устройству
- Нет механизма повторного подключения при сбоях
- Конфигурация может быть неполной

**Код в `application.ex`:**

```elixir
{SolarObservations.Workers.MountWorker,
 Application.get_env(:solar_observations, SolarObservations.Workers.MountWorker)},
```

**Решение**: Добавить валидацию конфигурации и механизм health-check.

### 2. Проблемы с расчетом траектории солнца

**Проблема 2.1**: В `solar_calculator.ex` используется упрощенная модель:

- Точность ~0.5°, что может быть недостаточно для точного слежения
- Модель склонения: `23.44 * sin(2π * (day_of_year - 81) / 365.25)`
- Не учитываются эффекты нутации, аберрации, уравнение времени

**Проблема 2.2**: Ошибка в строке 64:

```elixir
azimuth = 360.0 - azimuth  # Переменная переопределяется
```

**Решение**: Использовать более точные алгоритмы (например, алгоритмы из Meeus или VSOP87).

### 3. Проблемы с преобразованием координат

**Проблема**: В `mount_coordinates.ex` функция `radec_to_steps/4`:

- Преобразование RA в градусы: `ra / 24.0 * 360.0` - правильно
- Преобразование Dec: `dec + tube_offset` - может быть неправильным для отрицательных Dec
- Не учитывается прецессия, нутация

**Решение**: Добавить корректное преобразование с учетом всех астрономических поправок.

### 4. Проблемы с расчетом скорости слежения

**Проблема**: В `solar_tracker.ex` функция `calculate_t1_preset/4`:

- Формула может быть неправильной для high-speed режима
- Не проверяются граничные условия
- Используется солнечная скорость 15.041°/час, но реальная скорость меняется

**Текущая формула:**

```elixir
counts_per_sec = speed_deg_per_sec * cpr / 360.0
t1_preset_without_high_speed = trunc(timer_freq / counts_per_sec)
```

**Решение**: Проверить формулы из документации SkyWatcher и добавить валидацию.

### 5. Мелкие проблемы

1. Неиспользуемые переменные (предупреждения компилятора)
2. Отсутствие обработки ошибок в некоторых местах
3. Жестко закодированные значения (timeout, скорости)
